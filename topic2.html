Phase 1: The Blueprint (27 Minutes)

The Core Pillars of Database Architecture

A database isn't just a collection of spreadsheets; it‚Äôs a web of logic. To ensure your system doesn't collapse under data growth, we follow three main rules:

1. Normalization (Eliminating Redundancy)

  • 1NF: Atomic values (no lists in a single cell).
  • 2NF: No partial dependencies (everything relates to the Primary Key).
  • 3NF: No transitive dependencies (no "non-key" attributes depending on other "non-key" attributes).

2. Constraints & Integrity

Primary Keys: The unique DNA of a record.
Foreign Keys: The glue that connects tables.
Indexes: The "Table of Contents" that makes searches fast (O(log n) vs O(n)).

Phase 2: Building & Querying (63 Minutes)

Project: School Management System

We are going to build a schema that handles Employees, Projects, and their Assignments.

Step 1: Create the Tables (DDL)

Copy and paste this into your SQL editor to build the foundation.

-- Create Tables
CREATE TABLE Employees (
  EmpID SERIAL PRIMARY KEY,
  Name VARCHAR(100) NOT NULL,
  Salary DECIMAL(10,2)
);

CREATE TABLE Projects (
  ProjectID SERIAL PRIMARY KEY,
  ProjectName VARCHAR(100)
);

CREATE TABLE ProjectAssignments (
  AssignmentID SERIAL PRIMARY KEY,
  EmpID INT REFERENCES Employees(EmpID),
  ProjectID INT REFERENCES Projects(ProjectID),
  HoursWorked INT
);
